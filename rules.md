# Rules

These rules apply to every task in this repository.

## Core Order Of Operations

1. Read `rules.md`.
2. Check available skills and identify any skill that applies to the task.
3. Read and follow `GUARDRAIL.md` without exception.
4. Read `MEMORY.md` before starting work.
5. Update `MEMORY.md` during the task when durable project knowledge changes.
6. Update `MEMORY.md` again when the task is finished.

## Required Behavior

- Always follow `GUARDRAIL.md` before making decisions, edits, or claims.
- Always check available skills at the start of every task.
- Always use any relevant skill for the task before proceeding with substantial work.
- Always read `MEMORY.md` at the start of every task.
- Always update `MEMORY.md` when a task adds durable context, decisions, constraints, or follow-up items.
- Always update `MEMORY.md` after a task is completed with what changed and anything the next task should know.
- Keep changes scoped to the task.
- Preserve existing user work unless explicitly asked to change it.
- Verify important changes before reporting completion.

## Memory Update Standard

When updating `MEMORY.md`, prefer durable information over temporary notes:

- Current architecture or workflow decisions
- Constraints and conventions
- Known risks or recurring pitfalls
- Open follow-up items
- Short summaries of recently completed work

Do not fill `MEMORY.md` with noisy command logs or disposable scratch notes.
